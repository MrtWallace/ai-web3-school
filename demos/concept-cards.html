LLM
基于 Transformer 的神经网络，通过海量文本训练，学习语言模式。
  • 擅长：语言理解、代码生成、推理
  • 不擅长：精确事实记忆、确定性计算
AI 基础
点击翻转 →
LLM 工作原理
LLM 通过预测下一个 token 来生成文本。给定上下文，它计算所有可能 token 的概率分布，然后采样。

关键参数：
• temperature：控制随机性（0=确定，1=创意）
• max_tokens：最大输出长度
• top_p：核采样阈值
// API 调用示例
{
  model: "deepseek-chat",
  messages: [{role: "user", content: "Hello"}],
  temperature: 0.7
}
💬
Prompt
输入给 LLM 的文本，引导模型生成特定输出。
  • 上下文窗口：工作内存
  • 系统指令：身份和边界
  • 提示词：任务意图
  • 工具调用：从说话到做事
AI 基础
点击翻转 →
Prompt 的四个层面
1. 上下文窗口 — 模型能"看到"多少历史
2. 系统指令 — 设置身份、语气、边界
3. 用户提示 — 当前任务意图
4. 工具定义 — 模型能调用什么能力

好的 prompt = 明确的角色 + 清晰的任务 + 具体的约束
⚙️
Workflow
预定义的任务流程，模型是其中一个节点。
  • 特点：路径固定、可复现、可调试
  • 适用：流程固定、需要一致性的任务
AI 进阶
点击翻转 →
Workflow vs Agent
Workflow：路径固定，模型是处理节点
Agent：路径动态，模型自主决策

何时用 Workflow？
• 流程确定、不需要灵活应变
• 需要一致性和可审计性
• 错误处理路径可预定义
🧠
Agent
模型自主规划、动态调用工具、跨轮管理状态。
  • 状态管理、长期记忆、MCP
  • Skills、Tool Calling、Tracing
  • Guardrails、Handoff、错误恢复

AI Coding
用 AI 工具辅助编程：代码生成、调试、重构。
  • 工具：Claude Code、Cursor、Copilot
  • 擅长：样板代码、单元测试、文档
  • 不擅长：架构设计、业务逻辑决策

👛
钱包
管理私钥的工具，不是"存放"资产的地方。
  • 助记词 → 私钥 → 地址
  • 私钥 = 资产控制权
  • 泄露 = 资产丢失
Web3 基础
点击翻转 →
钱包安全要点
助记词：12/24 个单词，恢复钱包的唯一方式
私钥：控制资产的密钥，绝对不能泄露
地址：公开的收款地址，可以分享

⚠️ AI Agent 不应直接接触私钥！
→ 使用 Session Key 限制权限
✍️
签名
授权一个具体动作，不是"点一下确认"。
  • 签名 = 授权
  • 交易 = 执行
  • 不可逆、不可篡改
Web3 基础
点击翻转 →
签名 vs 交易
签名（Off-chain）
• 不消耗 Gas
• 证明你拥有私钥
• 常用于登录、授权

交易（On-chain）
• 消耗 Gas
• 改变链上状态
• 不可逆
⛽
Gas
链上执行的计算资源定价。
  • 防止滥用、激励矿工
  • 失败也会消耗
  • 价格由市场决定
Web3 基础
点击翻转 →
Gas 机制详解
Gas Limit：最大消耗量
Gas Price：每单位价格（Gwei）
总费用 = Gas Used × Gas Price

EIP-1559（以太坊）：
• Base Fee：基础费用（销毁）
• Priority Fee：小费（给矿工）

智能合约

  • 状态公开、执行公开
  • 升级权限可检查
  • 与后端的区别：无法随意修改

为什么 AI×Web3 需要 AA？
问题：AI Agent 不能直接持有私钥
解决：Session Key 给 Agent 临时权限

Session Key 可以限制：
• 只能调用特定合约
• 只能花固定金额
• 只在时间窗口内有效
• 过期自动失效
