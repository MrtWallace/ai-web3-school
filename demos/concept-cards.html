🤖
LLM
基于 Transformer 的神经网络，通过海量文本训练，学习语言模式。
  • 擅长：语言理解、代码生成、推理
  • 不擅长：精确事实记忆、确定性计算
AI 基础
点击翻转 →
LLM 工作原理
LLM 通过预测下一个 token 来生成文本。给定上下文，它计算所有可能 token 的概率分布，然后采样。

关键参数：
• temperature：控制随机性（0=确定，1=创意）
• max_tokens：最大输出长度
• top_p：核采样阈值
// API 调用示例
{
  model: "deepseek-chat",
  messages: [{role: "user", content: "Hello"}],
  temperature: 0.7
}
💬
Prompt
输入给 LLM 的文本，引导模型生成特定输出。
  • 上下文窗口：工作内存
  • 系统指令：身份和边界
  • 提示词：任务意图
  • 工具调用：从说话到做事
AI 基础
点击翻转 →
Prompt 的四个层面
1. 上下文窗口 — 模型能"看到"多少历史
2. 系统指令 — 设置身份、语气、边界
3. 用户提示 — 当前任务意图
4. 工具定义 — 模型能调用什么能力

好的 prompt = 明确的角色 + 清晰的任务 + 具体的约束
⚙️
Workflow
预定义的任务流程，模型是其中一个节点。
  • 特点：路径固定、可复现、可调试
  • 适用：流程固定、需要一致性的任务
AI 进阶
点击翻转 →
Workflow vs Agent
Workflow：路径固定，模型是处理节点
Agent：路径动态，模型自主决策

何时用 Workflow？
• 流程确定、不需要灵活应变
• 需要一致性和可审计性
• 错误处理路径可预定义
🧠
Agent
模型自主规划、动态调用工具、跨轮管理状态。
  • 状态管理、长期记忆、MCP
  • Skills、Tool Calling、Tracing
  • Guardrails、Handoff、错误恢复
AI 核心
点击翻转 →
Agent 核心组件
状态管理 — 跨轮保持上下文
Tool Calling — 调用外部工具
Guardrails — 防止越权
Memory — 长期记忆存储
Tracing — 执行链可视化

Sentinel 就是一个 Agent：解析意图 → 检查权限 → 执行交易
💻
AI Coding
用 AI 工具辅助编程：代码生成、调试、重构。
  • 工具：Claude Code、Cursor、Copilot
  • 擅长：样板代码、单元测试、文档
  • 不擅长：架构设计、业务逻辑决策

AI Coding 最佳实践
✅ 适合 AI 做：
• 生成样板代码和测试
• 解释陌生代码库
• 重构和代码风格统一

❌ 人必须做：
• 架构设计决策
• 业务逻辑定义

👛
钱包 Wallet
管理区块链密钥、与 dApp 交互的软件/硬件工具。
  • 资产永远在链上，钱包只存钥匙
  • MetaMask / OKX Wallet = Web3 数字网银
Web3 基础
点击翻转 →
钱包的本质
钱包 ≠ 存放资产的地方
钱包 = 管理私钥 + 展示资产 + 提供签名

⚠️ 常见误区：以为币"存在"钱包软件里
实际：资产在区块链上，钱包只有钥匙

💡 AI Agent 不应直接接触私钥
→ 使用 Session Key 限制权限
🔑
助记词 Seed Phrase
12-24 个随机英文单词，生成所有账户和私钥的"母钥匙"。
  • 输入任何兼容钱包即可恢复全部资产
  • 永远不要截屏、拍照或存联网设备
  • 最安全：线下手抄物理存储
Web3 安全
点击翻转 →
助记词 = 母钥匙
助记词 → 派生 → 所有私钥 → 所有地址

🚨 泄露 = 所有资产瞬间清零
黑客用自动化脚本几秒搬空

❌ 截屏 / 拍照 / 备忘录
✅ 手抄纸上，锁在保险柜
🔐
私钥 Private Key
64 位十六进制字符串，证明地址所有权并对交易签名。
  • 从助记词派生的"分钥匙"
  • 对应某一个具体地址
  • 私钥给出去 = 永久失去控制权
Web3 安全
点击翻转 →
私钥的真相
区块链只认私钥不认人
私钥 = 地址的唯一控制权

格式：0x4c2a...3f9e（64位十六进制）

🚨 借给朋友做测试网任务 = 永久失去
→ 没有"撤销"，没有客服
📍
地址 Address
由私钥计算生成的公开字符串，区块链上的"银行账号"。
  • 以太坊：0x 开头，42 字符
  • 可以公开分享，用于收款
  • 复制后务必核对前四位和后四位
Web3 基础
点击翻转 →
地址安全
地址 = 公开的收付款码
私钥 → 算法 → 地址（不可逆）

⚠️ 剪贴板恶意软件：
复制地址时悄悄替换成黑客地址
→ 复制后必须肉眼核对前后四位
✍️
签名 Signature
通过私钥证明"我确认并同意了某项操作"。
  • 签名（Off-chain）：不花 Gas，证明身份
  • 交易（On-chain）：花 Gas，改变链上状态
  • 签名 = 白纸黑字按指纹，不可伪造
Web3 安全
点击翻转 →
签名的危险
登录 Uniswap 弹出 "Sign Message"
→ 不花 Gas，只证明地址所有权

🚨 盲签（Blind Signing）极度危险：
钓鱼网站弹出乱码要求签名
→ 签了 = 把钱包控制权交出去

⚠️ 永远不要签你看不懂的东西
💸
交易 Transaction
在区块链上引起状态改变的行为：转账、调用合约、部署应用。
  • 必须广播到网络并被打包进区块
  • 失败也扣 Gas（算力已消耗）
  • 不可逆、不可篡改
Web3 基础
点击翻转 →
交易的本质
交易 ≠ 只是转账
交易 = 任何链上状态改变

类型：
• ETH 转账
• 调用合约函数
• 部署新合约
• Token approve

🚨 失败也扣 Gas！节点已执行计算
⛽
燃气费 Gas
支付给验证者的手续费，补偿处理交易和执行合约的算力。
  • 总费用 = Gas Used × Gas Price
  • EIP-1559：Base Fee（销毁）+ Priority Fee（小费）
  • 失败也扣 Gas（算力已消耗）
Web3 基础
点击翻转 →
Gas 常见误区
测试网用 Sepolia ETH（免费领）

🚨 Gas 设太低 = 交易卡死
没有节点愿意打包，挂起几小时甚至几天

🚨 Gas 设太高 = 浪费钱
建议用动态估算（EIP-1559）
📜
智能合约 Smart Contract
部署在链上、不可篡改且自动执行的代码程序。
  • 满足触发条件 → 严格执行预设结果
  • 状态公开、执行公开、可审计
  • 与后端的区别：部署后无法随意修改
Web3 核心
点击翻转 →
合约的两面
例：存 1 ETH 满 30 天 → 自动释放 + 5% 利息

Sentinel SmartAccount.sol：
• Owner + Agent 双角色
• 每日支出限额
• 地址：0x3350...4074

🚨 代码不可篡改，但人会犯错
未审计合约 = 可能有漏洞/后门（Rug Pull）
🧪
测试网 Testnet
供开发者测试功能的实验性区块链，逻辑与主网一致但无真实价值。
  • 以太坊：Sepolia / Holesky
  • 测试币通过 Faucet 免费领取
  • 学习和实验应先在测试网完成
Web3 基础
点击翻转 →
测试网注意事项
Sentinel 部署在 Sepolia 测试网

🚨 测试币不能变现！
骗子说"高价出售测试币"= 诈骗
测试币只通过 Faucet 免费领取

⚠️
授权 Approval
与合约交互前，先授予合约"划走代币的额度"。
  • 类似给商户开"副卡额度"
  • Unlimited Approval = 无限额度
  • 恶意合约可随时刷光授权代币
Web3 安全
点击翻转 →
授权的陷阱
卖币前必须先 approve 合约
→ 合约获得"划走额度"

🚨 Unlimited Approval 危险：
恶意合约随时可以刷光所有该代币
不需要再次确认！

✅ 只授权需要的数量
✅ 用完立刻 revoke

为什么 AI×Web3 需要 AA？
问题：AI Agent 不能直接持有私钥
解决：Session Key 给 Agent 临时权限

Session Key 可以限制：
• 只能调用特定合约
• 只能花固定金额
• 只在时间窗口内有效
• 过期自动失效

Guardrails
防止 AI Agent 越权操作。
  • 黑名单检查
  • 阈值确认
  • Human-in-the-loop
