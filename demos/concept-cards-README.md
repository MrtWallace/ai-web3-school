## 1. 它解决什么学习问题

Week 1 要同时理解 AI 和 Web3 两套体系的概念（LLM、Agent、Gas、智能合约等），传统笔记有两个问题：

- **信息平铺**：12 个概念堆在一起，找不到重点，看不出分类关系
- **单向阅读**：看完了记不住，没有主动回忆的交互

概念卡片解决这两个问题：

- **分类可视化**：AI（蓝）、Web3（粉）、AI×Web3（青）三色区分，一眼看出哪边概念多、哪边薄弱
- **翻转交互**：正面是概览（3 秒抓住核心），背面是深度解释 + 实际例子（需要时再翻）
- **搜索筛选**：快速定位某个概念，或按类别集中复习

## 2. 用户如何与它交互

| 操作 | 效果 |
|------|------|
| **点击卡片** | 正面 ↔ 背面翻转（3D 动画） |
| **搜索框输入** | 实时过滤，只显示匹配的卡片 |
| **点击分类按钮** | 只看 AI / Web3 / AI×Web3 其中一类 |
| **翻页滚动** | 移动端自适应布局 |

交互流程：
```
打开页面 → 看到 12 张卡片 + 统计数字
→ 点击感兴趣的卡片 → 翻转看到深度解释
→ 搜索 "agent" → 只显示 Agent 相关卡片
→ 点击 "AI" 按钮 → 只看 AI 类概念
```

## 4. 哪部分由 AI 生成，哪部分由人工修改 / 验证

### AI 生成（Hermes Agent, mimo-v2.5-pro）

| 内容 | AI 贡献 |
|------|---------|
| HTML 结构 | 生成完整页面框架、卡片组件、翻转动画 CSS |
| JavaScript | 搜索过滤、分类筛选、统计更新逻辑 |
| 概念内容初稿 | 12 张卡片的正面文字（从 `concepts.md` 提取） |
| 背面深度解释 | API 参数、组件列表、架构图等 |

### 人工修改 / 验证

| 内容 | 人工修改 |
|------|---------|
| Sentinel 实例 | 背面内容用 Sentinel 真实数据替换 AI 的泛化描述 |
| 合约地址 | 填入真实部署地址 `0x3350...4074` |
| CSS 调试 | 修复背面文字溢出（减小字体、加滚动条） |
| 关键词索引 | 手动添加 `data-keywords` 用于中文搜索 |
| 数据准确性 | 验证每个概念的解释与 `concepts.md` 一致 |

### 比例估计
- AI 生成：~70%（结构、样式、初稿内容）
- 人工修改：~30%（真实数据、调试、验证）

## 5. 还有哪些限制或下一步改进

### 当前限制

| 限制 | 说明 |
|------|------|
| **纯静态** | 没有后端，数据硬编码在 HTML 里，增删概念要改代码 |
| **无记忆** | 刷新页面后搜索和筛选状态丢失 |
| **单语言** | 只有中文，没有英文切换 |
| **无进度追踪** | 不记录用户看了哪些卡片、哪些已掌握 |
| **移动端体验** | 小屏上卡片堆叠，需要大量滚动 |

### 下一步改进

| 优先级 | 改进项 | 预期效果 |
|--------|--------|---------|
| P1 | 加"已掌握/未掌握"标记 | 用户能追踪学习进度 |
| P2 | 数据驱动（JSON 配置） | 增删概念不用改 HTML |
| P3 | 加 quiz 模式 | 翻转后显示测试题，主动回忆 |
| P4 | 英文切换 | 方便向国际社区展示 |
| P5 | 与 concepts.md 联动 | 自动从 markdown 生成卡片数据 |
