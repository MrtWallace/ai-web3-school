# Week 1 — 交叉实验

## Sentinel 项目：AI 输出到链上执行的完整流程

### 流程图

```
┌─────────────────────────────────────────────────────────┐
│                    用户输入（自然语言）                    │
│                   "Swap 0.001 ETH to USDC"               │
└─────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────┐
│              AI 意图解析（DeepSeek API）                  │
│    - 解析意图: swap                                       │
│    - 提取参数: amount=0.001, token_in=ETH, token_out=USDC │
│    - 耗时: ~1.5s, 成本: ~$0.00004/call                   │
└─────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────┐
│              Guardrails 检查（链下）                      │
│    - 黑名单检查: 目标地址是否在黑名单                      │
│    - 阈值检查: 金额是否超过每日限额                       │
│    - 如果触发 → 暂停，请求人工确认                        │
└─────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────┐
│              人工确认节点（可选）                          │
│    - 当金额超过阈值时，请求用户确认                        │
│    - 当地址在黑名单时，直接拒绝                            │
└─────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────┐
│              钱包签名确认                                 │
│    - 用户确认交易详情                                     │
│    - 签名授权                                             │
└─────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────┐
│              链上执行（SmartAccount.execute()）            │
│    - 调用 Uniswap V3 exactInputSingle                    │
│    - Gas 消耗: ~XXX                                       │
│    - 状态: 成功/失败                                      │
└─────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────┐
│              区块浏览器验证                                │
│    - 交易哈希: 0x...                                      │
│    - 区块高度: XXX                                        │
│    - 状态: Success                                        │
│    - 链接: https://sepolia.etherscan.io/tx/0x...         │
└─────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────┐
│              日志记录                                     │
│    - 意图解析结果                                         │
│    - Guardrails 检查结果                                  │
│    - 执行结果                                             │
│    - 耗时和成本                                           │
└─────────────────────────────────────────────────────────┘
```

### 关键节点分析

| 节点 | 类型 | 失败点 | 回滚策略 | 人工确认 |
|:--|:--|:--|:--|:--|
| 意图解析 | AI | 解析错误、幻觉 | 重试（已有 retry 机制） | 可选 |
| Guardrails | 规则 | 规则遗漏 | 拒绝执行 | 触发时需要 |
| 钱包签名 | 用户 | 用户拒绝 | 取消交易 | 必须 |
| 链上执行 | 合约 | Gas 不足、合约错误 | 交易回滚 | 不可逆 |
| 验证 | 链上 | 网络延迟 | 等待确认 | 无需 |

### 三种方式对比

| 维度 | 纯人工 | AI 辅助 | Sentinel 自动化 |
|:--|:--|:--|:--|
| **速度** | 慢（手动操作） | 中（AI 解析 + 人工执行） | 快（全自动） |
| **成本** | 低 | 中（API 调用） | 中（API + Gas） |
| **风险** | 低（人工控制） | 中（AI 可能出错） | 高（需要 guardrails） |
| **可扩展性** | 低 | 中 | 高 |
| **适用场景** | 高价值交易 | 日常操作 | 批量/自动化操作 |

### 安全边界

**AI Agent 绝对不能自动执行的操作**:
- 🔴 私钥/助记词 — 永远不要输入、粘贴或让 agent 读取
- 🔴 API Key 泄露 — 不要把真实 API key 写入代码或 commit
- 🔴 签名操作 — 任何需要钱包签名的操作必须人工确认
- 🔴 转账/交易 — 涉及真实资产的交易必须人工确认
- 🔴 合约部署到主网 — 只允许测试网部署
