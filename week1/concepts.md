# Week 1 — 基础概念说明

> 本文件记录 AI × Web3 核心概念的理解，作为后续学习的基础。

## AI 方向（7 个概念）

### LLM（Large Language Model，大语言模型）
- **一句话解释**: 通过数据训练过的、基于 Transformer 架构的深度神经网络。
- **例子**: DeepSeek V4、GPT-5.5、Gemini 3.1 这些模型都是 LLM。
- **常见误区**: 把公司和模型弄混。比如 OpenAI ≠ ChatGPT（OpenAI 是公司，ChatGPT 是产品），Claude ≠ Opus 4.6（Claude 是 Anthropic 的产品线，Opus 是其中一个模型版本）。

### Prompt（提示词）
- **一句话解释**: 输入给 LLM 的文本，引导模型生成特定输出。设置身份、语气和边界（例如"你是一个 xx 方面的专家"这种）。
- **核心作用**: 约束 AI 回答的概率空间，使回复更容易落到期望的范围。
- **常见误区**: 以为 prompt 只是"问问题"，实际上 prompt 是在控制模型的输出分布——越精确的 prompt，输出越可控。

### Context Window（上下文窗口）
- **一句话解释**: 存储历史会话 + 当前输入 + 模型输出的地方。之前设定的身份、语气这些 prompt 不会在下一个对话就忘记的关键。
- **例子**: 根据模型不同有大小区别（128K / 256K / 1M tokens）。
- **常见误区**: 以为模型"记住了"之前的对话——实际上它只是在 context window 范围内能看到历史，超出窗口的内容就丢失了。

### Workflow（工作流）
- **一句话解释**: 预定义的任务流程，模型是其中一个节点。
- **例子**: 自动处理数据到 Excel（输入 → 模型 → Excel → 输出），模型只是在中间参与一个节点，输入输出的判断、出现异常情况如何处理还是需要人来解决。
- **常见误区**: 以为加了 LLM 就是"智能"了——实际上 workflow 的路径是固定的，模型只是其中一个处理环节，不具备自主决策能力。

### Agent（智能体）
- **一句话解释**: 模型自主规划、动态调用工具、跨轮管理状态。
- **与 Workflow 的区别**: Agent 比 Workflow 多了模型自主规划、获取任务状态、检查中间结果并调整的能力。
- **常见误区**: 以为 Agent 什么都能自动做——实际上 Agent 需要明确的工具边界和 human-in-the-loop 机制来防止越权。

### Tool Use（工具调用）
- **一句话解释**: 调用外部能力：搜索、数据库、API、代码执行、邮件、支付接口、内部系统等。
- **核心价值**: 从"对话窗口"扩展到更多的使用场景，让模型从"说话"变为"做事"。
- **常见误区**: 以为模型自己会用工具——实际上工具调用是框架层面的事，模型只是输出结构化请求（如 JSON），框架负责执行并回传结果。

### Human-in-the-Loop（人机协同）
- **一句话解释**: Agent 每做一步关键决策，都必须经过人确认。
- **工作方式**: 当运行到指定节点（例如 tools 节点）之前，会强制挂起（Suspend），并将当前状态保存到 Checkpoint 中。此时程序退出，等待下一次唤醒。
- **常见误区**: 以为 human-in-the-loop 会拖慢效率——实际上在涉及资产、权限、签名的场景中，这是必须的安全机制，不是可选项。

## Web3 方向

### 账户、地址和钱包
- **账户**: 链上身份，由私钥控制
- **地址**: 账户的公开标识（公钥的哈希）
- **钱包**: 管理私钥的工具，不是"存放"资产的地方
- **关键**: 私钥 = 资产控制权，泄露 = 资产丢失

### 助记词、私钥、地址
- **助记词**: 12/24 个单词，生成私钥的种子
- **私钥**: 256 位随机数，控制账户
- **地址**: 从私钥推导出的公开标识
- **安全底线**: 助记词和私钥永远不能泄露、输入、粘贴或让 agent 读取

### 签名与交易
- **签名**: 授权一个具体动作，不是"点一下确认"
- **交易**: 签名后的指令，广播到网络执行
- **关键**: 签名 = 授权，交易 = 执行

### Gas
- **是什么**: 链上执行的计算资源定价
- **为什么**: 防止滥用、激励矿工/验证者
- **特点**: 失败也会消耗 Gas、价格由市场决定

### 智能合约
- **是什么**: 部署在链上的程序，自动执行预定义逻辑
- **特点**: 状态公开、执行公开、升级权限可检查
- **与后端的区别**: 无法随意修改、执行成本高、透明度高

### 主网与测试网
- **主网**: 真实资产、真实成本
- **测试网**: 测试用、免费测试币、无真实价值
- **原则**: 学习和实验应先在测试网完成

### 区块浏览器
- **是什么**: 查看链上数据的工具（如 Etherscan）
- **用途**: 查看交易状态、合约代码、账户余额

## AI × Web3 交叉

### 为什么交叉是趋势
- AI 擅长理解意图、生成内容、做出决策
- Web3 擅长去信任执行、资产控制、透明记录
- 交叉点：AI Agent 自主执行链上操作、可验证 AI 推理、链上数据智能分析

### 关键基础设施
- **账户抽象 (ERC-4337)**: 让 AI Agent 可以管理智能账户
- **Session Key**: 限制 AI Agent 的权限范围
- **Guardrails**: 防止 AI Agent 越权操作
